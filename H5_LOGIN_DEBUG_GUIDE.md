# 🔍 H5登录问题排查指南

## 问题描述
用户反馈：页面进入时有权限提示框，允许后点击登录无效。

---

## 📋 排查步骤

### 步骤1：打开浏览器控制台

**操作**：
- 按F12打开开发者工具
- 切换到"Console"标签

**查看内容**：
- 有没有红色错误信息？
- 有没有黄色警告信息？
- 有没有登录相关的日志输出？

---

### 步骤2：检查网络请求

**操作**：
- 按F12打开开发者工具
- 切换到"Network"标签
- 点击登录按钮
- 查看是否有新的请求

**预期结果**：
应该看到：
```
POST https://marketing-backend-cu2q.onrender.com/api/v1/auth/verify
Status: 200 OK
Response: {"success":true,"valid":true,"message":"授权验证成功"}
```

**如果没有请求**：
- 点击按钮没有触发
- 可能是按钮事件未绑定

**如果请求失败**：
- Status: 404, 500, 或其他错误码
- 查看错误详情

---

### 步骤3：查看控制台日志

**预期的日志流程**：

```
=== 开始登录流程 ===
输入 - 店面编号: test
输入 - 店面名称: test
输入 - 登录密码: ***
验证通过，开始登录请求...
后端URL: https://marketing-backend-cu2q.onrender.com
请求URL: https://marketing-backend-cu2q.onrender.com/api/v1/auth/verify
响应状态: 200 OK
登录响应数据: { success: true, valid: true, message: '授权验证成功' }
登录成功，保存到本地存储
登录结果: true
用户点击确定，跳转到首页
```

**如果看不到日志**：
- JavaScript加载失败
- 页面未正确渲染

---

### 步骤4：检查权限提示

**可能的权限提示**：

1. **通知权限**
   - 浏览器询问："是否允许发送通知？"
   - 这不会影响登录功能

2. **地理位置权限**
   - 浏览器询问："是否允许获取地理位置？"
   - 这不会影响登录功能

3. **IndexedDB/存储权限**
   - 浏览器询问："是否允许使用存储？"
   - 这**可能**影响登录（需要存储登录状态）

4. **其他权限**
   - 摄像头、麦克风等
   - 这些不应该出现

**如果提示存储权限**：
- ✅ 点击"允许"
- 如果不允许，登录状态无法保存，但应该能登录一次

---

### 步骤5：清除缓存和Cookie

**操作**：
- F12打开开发者工具
- 右键点击刷新按钮
- 选择"清空缓存并硬性重新加载"

**或者**：
- Chrome: Ctrl+Shift+Delete
- Firefox: Ctrl+Shift+Delete
- Safari: Cmd+Option+E

---

## 🔧 常见问题与解决方案

### 问题1：点击登录无任何反应

**原因**：按钮事件未绑定或JavaScript错误

**解决方案**：
1. 查看控制台是否有JavaScript错误
2. 清除浏览器缓存
3. 使用无痕模式测试

---

### 问题2：提示"网络错误，请重试"

**原因**：无法连接到后端

**解决方案**：
1. 检查网络连接
2. 测试后端服务：
   ```bash
   curl https://marketing-backend-cu2q.onrender.com/api/v1/health
   ```
3. 查看Network标签中的请求详情

---

### 问题3：提示"授权码错误"

**原因**：授权码不正确

**解决方案**：
1. 使用正确的测试账号：
   - 店面编号：`test`
   - 店面名称：`test`
   - 登录密码：`test` 或 `lenovoxinan`
2. 检查输入是否有多余空格

---

### 问题4：可以登录但刷新后需要重新登录

**原因**：登录状态无法保存（存储权限被拒绝）

**解决方案**：
1. 允许浏览器使用存储权限
2. 检查浏览器是否禁用Cookie和站点数据
3. 尝试其他浏览器

---

### 问题5：页面空白或白屏

**原因**：JavaScript加载失败

**解决方案**：
1. 查看控制台错误
2. 检查网络连接
3. 确认H5已正确部署
